│                              DATA FLOW SEQUENCE                                │
└─────────────────────────────────────────────────────────────────────────────────┘

1. USER ACTION:          Search input / Filter selection / Page click
                         │
                         ▼
2. EVENT HANDLER:        handleSearch() / handleFilter() / onPageChange()
                         │
                         ▼
3. REDUX DISPATCH:       dispatch(setSearchTerm/setFilterCategory/setCurrentPage)
                         │
                         ▼
4. STATE UPDATE:         Redux store updates state
                         │
                         ▼
5. RE-RENDER:            useSelector() triggers component re-render
                         │
                         ▼
6. COMPUTATION:          filteredItems → currentItems → totalPages
                         │
                         ▼
7. UI UPDATE:            New items displayed, pagination updated

┌─────────────────────────────────────────────────────────────────────────────────┐
│                              KEY FEATURES                                      │
└─────────────────────────────────────────────────────────────────────────────────┘

• SEARCH:     Real-time filtering by item name (case-insensitive)
• FILTER:     Category-based filtering (All/Fruit/Vegetable)
• PAGINATION: Page-based navigation with configurable items per page
• STATE MGMT: Centralized Redux state management
• AUTO-RESET: Page resets to 1 when search/filter changes
